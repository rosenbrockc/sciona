Walkthrough A: ECG Signal Processing Pipeline

This walkthrough demonstrates automated design and verification of cardiac signal processing filters using biosppy-compatible atoms.

Step 1: Signal Ingestion
A high-frequency ECG signal is loaded. Notice the raw signal contains significant muscle tremor and baseline drift.
Step 2: Bandpass Filtering
A Butterworth bandpass filter (0.5 to 40 Hz) is applied. If the input domain does not match (e.g. frequency domain fed into time domain), GhostSim raises a mismatch.
Step 3: R-Peak Segmentation
Hamilton segmentation detects peak intervals. Click Run Node to verify inputs/outputs incrementally.

Walkthrough B: Tabular Machine Learning

This walkthrough illustrates tabular ML model selection, training, and ensembling. Learn how to resolve ungrounded estimators using the Delta Planner.

Step 1: Base Pipeline
A basic estimator (e.g. simple regression or classification fit) is initially constructed. This has low validation split coverage (0.50).
Step 2: Quick-Fix Ensembling
Delta Planner recommends apply_kfold_ensemble and apply_stacking_ensemble. Click "Apply" in the sidebar to expand the graph into a robust ensemble pipeline.

Walkthrough C: UMAP Scientific Computing

This walkthrough shows how complex dimensionality reduction (PCA, t-SNE, UMAP) is structured, composed, and validated.

Step 1: High-Dimensional Features
High-dimensional dataset is passed through initial data assembly.
Step 2: Dimensionality Reduction
UMAP projection maps the space into 2D. We verify spectral stability and layout constraints using GhostSim witnesses.
